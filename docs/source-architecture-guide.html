4. 모방학습 & 강화학습 (상세)

본 프로젝트는 2단계 계층 RL + 휴리스틱 Teacher 모방학습(BC) + PPO 미세조정 구조입니다. 라이브러리: stable-baselines3(상위 PPO), sb3-contrib(하위 MaskablePPO).

train_model(problems)
  │
  ├─ [Phase 0] shape 필터: (obs_shape, action_n) 동일 문제만
  │
  ├─ [Phase A] train_alloc_model()  ─ USE_ALLOC_MODEL ∧ ALLOC_LAMBDA>0
  │     BC(MSE→해석식 logit) → PPO → ppo_alloc.zip
  │     역할: Mode1 가이드 (공정×모델 목표 장비 대수)
  │
  └─ [Phase B] MaskablePPO dispatch
        collect_teacher_dataset(heuristic) → BC(마스킹 CE) → PPO → ppo_dispatch.zip
        역할: Mode2 시간당 장비 이동 (실시간 배치)

4.1 Teacher — heuristic_actions (모방학습 레이블)

simulator.heuristic_actions(sim, s)는 BC 데이터의 “정답 행동”을 만듭니다. Oracle이 아니라 그리디 휴리스틱입니다.

입력Simulator, SimState (현재 시간·배치·WIP·idle)
알고리즘
  1. valid_moves(s) 후보 중, 이번 1시간에 잔여계획을 가장 많이 줄이는 Move 1개 선택
  2. gain = min(to_rem, wip[to], UPH[to] × 남은시간) — batch 전환 시 switch_time 차감
  3. from이 아직 생산 가능(from_rem>0, wip>0)이면: 같은 batch 무료 이동만 허용 (UPH 개선 or 빈 task 충원)
  4. 최대 sum(eqp_qty)+1회 반복 → substep 이동 리스트
출력list[Move] — DispatchEnv에서 action 1..N으로 매핑 후 마지막에 action 0(commit)

4.2 상위 — AllocationEnv (Mode 1 가이드)

환경 특성

에피소드1 step — reset → step → terminated (horizon 동안 고정 배분 결정)
관측 (Box [0,1]) dim = mt + 2·mm·mt + mm  (mt=MAX_TASKS, mm=MAX_MODELS)
① plan_qty 정규화 × mt
② UPH 행렬 / max_uph × mm·mt
③ eqp_qty / max_eqp × mm
④ tool_qty / max_tool × mm·mt (batch×model)
액션 (Box [-3,3])shape = mm×mt — 모델별 task logit (연속)
내부 변환 model별 softmax → eqp_qty 비례 raw 배분 → tool 상한 clip → 합계 eqp_qty 스케일 → 최대잉여법(largest remainder)으로 정수화 → alloc[(model,ti)]
보상task별 이론 계획달성률 평균
cap = Σ alloc[m,t]×UPH[m,t]
switches_in = max(0, alloc − init_assign) 합
eff_h = horizon − switches_in × switch_time_hours
rate_t = min(cap×eff_h, plan_qty) / plan_qty
reward = mean(rate_t)
출력 (추론)get_float_target() → complete_guide_allocation → DispatchEnv target_allocation

모방학습 — behavior_clone_alloc

Teacherplan_target_allocation() 해석식 (plan_qty/UPH 비례, eqp_qty 스케일)
타깃 logit_analytic_target_logits(env): frac = analytic/(model,ti)/eqp_qty → log(frac)+3, clip [-3,3], UPH 없는 슬롯 -3
손실MSE( policy.mean − target ) × mask / sum(mask) — UPH 있는 (model,task)만
옵티마Adam, lr=BC_LR(1e-3), epochs=BC_EPOCHS(300)
데이터문제당 obs 1개 (1-step env) — problems 리스트 전체

강화학습 — train_alloc_model PPO

알고리즘sb3.PPO("MlpPolicy") (Maskable 아님 — 연속 Box action)
하이퍼n_steps=64, batch_size=32, total_timesteps=max(2000, ppo_steps//10)
env 샘플링매 rollout마다 random.choice(same_shape_problems)
저장saved_models/ppo_alloc.zip

4.3 하위 — DispatchEnv + MaskablePPO (Mode 2 동적 배치)

시간 구조 (에피소드)

1 시간 = 여러 substep(장비 Move) + commit(action=0)
  substep: apply_move (생산 없음, idle/tool 상태만 변화)
  commit: advance_hour() → UPH×배치 생산, WIP 유입, hour++
  max_substeps = sum(eqp_qty) + 1  (초과 시 강제 commit)

관측 벡터 (DispatchEnv._obs)

기본 dim = 2·mt + 2·mm·mt + 1. DWELL_OBS=true이면 +mt (체류시간 슬롯).

슬롯크기의미 · 정규화
plan_partmt잔여계획 / plan_qty (task별)
wip_partmt현재 WIP / max(init_wip)
assign_partmm×mt배치대수 / eqp_qty[model]
change_partmm×mtidle(전환중) / (switch_time×eqp_qty) — 목적지 task 인코딩
hour_part1hour / horizon_hours
dwell_partmt (옵션)wip_dwell_time / horizon (WIP 체류 추정)

예: mt=5, mm=3, DWELL_OBS=true → obs_dim = 10 + 30 + 30 + 1 + 5 = 76

액션 공간 & 마스킹

action 0commit — 1시간 경과 (항상 mask=True)
action 1..N사전 열거 Move(model, from, to) — padded mm×mt×(mt−1) 조합
마스크sim.valid_moves(s)에 있는 Move만 True (tool·conv_group·UPH·assign 검사)
래퍼ActionMasker(env, _mask_fn) → MaskablePPO 호환

보상 함수 (step마다 dense + 종료 보너스)

reward = (achievement_gain / total_plan) + dwell_r + alloc_r  [+ terminal bonus]

achievement_gain = Δ min(produced, plan_qty) 합  (commit/substep 후 생산 증가분)

dwell_r (commit 시, DWELL_LAMBDA>0):
  Σ weight_i × min(dwell_i, H)/H
  weight_i = remaining_plan_i / total_plan

alloc_r (commit 시, ALLOC_LAMBDA>0, target_allocation 있음):
  가동률 ≥ GUIDE_UTIL_THRESHOLD 일 때만
  task별 WIP>0 슬롯: assign이 target±GUIDE_BAND_PCT 밴드 밖이면 페널티
  alloc_r = ALLOC_LAMBDA × (1 − mean_penalty)

terminated 시: reward += plan_achievement  (에피소드 종료 보너스)

모방학습 — collect_teacher_dataset + behavior_clone

데이터 수집 문제마다 DispatchEnv + 동일 target_allocation(Phase A 결과)
매 substep: obs 저장 → teacher move를 action index로 변환 → env.step
invalid action이면 0(commit)으로 대체
guard: horizon×(eqp+2)+5 로 무한루프 방지
출력 텐서obs float32[], acts int[], masks bool[] — 전 문제·전 substep 누적
BC 손실CrossEntropy( masked_logits, teacher_action )
invalid action logit = −1e8 (mask False)
옵티마Adam lr=BC_LR, BC_EPOCHS=300, 전체 배치 1-pass/epoch

강화학습 — MaskablePPO (train_model)

알고리즘MaskablePPO("MlpPolicy") + ActionMasker
하이퍼n_steps=256, batch_size=64, total_timesteps=DEFAULT_PPO_STEPS(50_000) 또는 CLI --steps
envmake_env(random.choice(problems)) — 매 rollout 다른 ProblemInstance
순서① BC 사전학습 ② model.learn(ppo_steps) on-policy 미세조정
저장saved_models/ppo_dispatch.zip

4.5 학습 vs 추론 차이

항목학습 (train.py)추론 (test.py / infer)
보상PPO/BC loss 계산에 사용사용 안 함 — Simulator만 실행
정책stochastic rollout (PPO)predict(..., deterministic=True)
가이드학습 중 ppo_alloc 또는 해석식 → target_allocation 주입_guide_allocation() → RTS_GUIDE rows
RL 실행env.step 루프로 학습_rl_policy_factory: sim 상태를 env에 주입, substep predict 후 sim.apply_move

alloc BC teacherplan_target_allocation 해석식동일 (추론 시 RL alloc 모델 predict)
dispatch BC teacherheuristic_actions추론 미사용 (학습 전용)

추론 시 RL policy 루프 (_rl_policy_factory)

매 시뮬레이션 1시간:
  env._state = sim state (공유)
  moves = []
  repeat up to max_substeps:
    obs = env._obs()
    mask = env.action_masks()
    action = model.predict(obs, action_masks=mask, deterministic=True)
    if action==0: break
    mv = move_list[action-1]
    if mv valid: sim.apply_move(s, mv); moves.append(mv)
    else: break
  return moves  → run_policy가 advance_hour 전까지 적용

4.6 MDP 정식 정의 (강화학습 관점)

본 프로젝트는 계층 MDP입니다. 고정 파라미터 ξ = ProblemInstance 위에서 두 개의 Gym 환경이 순차 실행됩니다. 실제 물리 상태는 SimState이고, 정책이 보는 것은 정규화된 관측 o = f(s, ξ)입니다.

계층 구조 (에피소드 1개 = ProblemInstance ξ 하나)

  [상위 MDP] AllocationEnv  — 1 transition 후 종료
       │  출력: target_allocation (가이드) + RTS_GUIDE rows
       ▼
  [하위 MDP] DispatchEnv     — horizon×(substeps) transitions
       │  내부: Simulator SimState s
       ▼
  산출: trace, hourly_stats → PLAN_ACHV / ASSIGN / CONV rows

상위 MDP — AllocationEnv (Mode 1)

요소정의
S (실질)정적 문제 파라미터만 — ξ (plan, UPH, eqp, tool). 에피소드 중 변하지 않음.
O (관측)o ∈ [0,1]^{mt + 2·mm·mt + mm} — plan, UPH행렬, eqp, tool 정규화 (_build_obs)
A (행동)a ∈ [-3,3]^{mm×mt} 연속 logit. 내부 결정론적 디코더 g(a)→alloc ∈ ℤ^{mm×mt} (softmax→tool clip→largest remainder)
T (전이)1-step terminal. step(a) 후 done=True. SimState 없음 — 이론 생산량만 계산.
R (보상)R = mean_t min(cap_t·eff_h, plan_t) / plan_t
cap_t = Σ_m alloc[m,t]·UPH[m,t]
eff_h = H − (Σ_m max(0,alloc−init_assign))·switch_time
γ사실상 1 (1-step) — SB3 PPO 기본 γ≈0.99 적용되나 에피소드 길이=1
정책학습: π_alloc(a|o) Gaussian (PPO). 추론: a = π(o) deterministic → guide rows

하위 MDP — DispatchEnv (Mode 2)

요소정의
S (hidden)s = SimState(hour, produced, wip, assign, idle, tool_used) — Simulator가 관리
O (관측)o = _obs(s) ∈ [0,1]^d, d = 2·mt + 2·mm·mt + 1 (+mt if DWELL_OBS). POMDP에 가까움 — idle·전환 의도가 change_part에 부분 노출.
A (행동)이산 a ∈ {0,1,…,|move_list|}. 0=commit(1h 경과), k>0=미리 열거된 Move.
마스크M(s) ⊂ A: commit 항상 True + valid_moves(s) (tool·conv_group·UPH·assign>0)
T (전이) • a=0 또는 substep 한도: s' = advance_hour(s) — 생산·WIP유입·hour++
• a=k valid move: s' = apply_move(s, Move_k) — assign/idle/tool 변화, 생산 없음
• invalid move: 상태 유지 (substep만 증가)
R (보상, 학습만)매 env.step:
r = Δachievement/total_plan + dwell_r·𝟙_commit + alloc_r·𝟙_commit
종료 시: r += plan_achievement(s)
종료hour ≥ horizon_hours → terminated
옵션 구조1 “거시 step” = substep들(move) + commit. BC/PPO는 미시 step마다 (o,a,r) 기록.

benchmark_01에서의 MDP 스냅샷 (실측)

tSimState (요약)valid actions대표 aΔproduced
reseth=0, wip=1000, assign M1→t0=1mask True=1 (commit만)a=0—
commit h0→1h=1, produced=100, wip=900동일a=0+100
commit h1→2h=2, produced=200동일a=0+100
commit h2→3h=3, produced=300, done—a=0+100, R_terminal=1.0

obs_dim=46 (mt=5,mm=3,DWELL_OBS), action_n=61. 이동 없으면 매 micro-step이 commit만 반복되는 degenerate case.

4.7 MDP 기준 데이터 변경 파이프라인

“데이터”는 세 층으로 흐릅니다: 정적 ξ → MDP 전이 (s,o,a,r) → 운영 산출 (Oracle rows).

A. 학습 (train) — MDP rollout → 정책 갱신

[정적 입력 ξ]
  DB/JSON → ProblemInstance
  필드: tasks[], uph{}, eqp_qty{}, init_assign{}, tool_qty{}, horizon, switch_time
  + config.CONV_GROUPS (ξ에 주입)

[상위 MDP — AllocationEnv]  (선택, USE_ALLOC_MODEL)
  o_alloc = _build_obs(ξ)                    … 고정 특징 벡터
  BC: target a* = analytic_logit(plan_target_allocation(ξ))
  PPO rollout: (o_alloc, a, r_alloc) × N problems
  g(a) → alloc → complete_guide_allocation → target_allocation (ξ에 결합, dispatch용)

[하위 MDP — DispatchEnv + MaskablePPO]
  reset: s₀ ~ T_reset(ξ), o₀ = _obs(s₀)
  BC dataset (teacher π_H = heuristic):
    repeat until done:
      mask_t = M(s_t)
      a_t = index(π_H(s_t))  … move indices + commit
      store (o_t, a_t, mask_t)
      (s_{t+1}, r_t) = env.step(a_t)   … Simulator 전이 + reward

  PPO learn:
    rollouts: (o_t, a_t, mask_t, r_t, done) ~ π_θ
    advantage / policy gradient → θ update
  save π_θ → ppo_dispatch.zip

[학습 산출물 — MDP 밖]
  ppo_alloc.zip, ppo_dispatch.zip (정책 파라미터 θ)

B. 추론 (infer) — MDP 전이는 실행, 보상·역전파 없음

[정적 입력 ξ]  (train과 동일 경로)
  Oracle fetch_rows → rows_to_problem → save/load JSON

[상위 — 가이드 only, reward 없음]
  o_alloc = _build_obs(ξ)
  a = π_alloc(o) deterministic  (또는 analytic fallback)
  alloc = g(a)
  guide_rows = build_guide_rows(complete_guide_allocation(alloc))
  target_allocation → DispatchEnv 생성 시 주입 (ALLOC_LAMBDA>0)

[하위 — 시뮬레이션 두 경로]

  (1) Heuristic path (항상):
    s ← reset(ξ)
    each hour h:
      moves = heuristic_actions(sim, s)   … MDP 밖 teacher, sim 직접 조작
      apply moves on sim
      advance_hour(s)                   … T_commit
    → trace_h, hourly_stats_h

  (2) RL path (shape match 시):
    s ← reset(ξ)  (sim에 주입)
    each hour:
      repeat micro-step:
        o = _obs(s), mask = M(s)
        a = π_dispatch(o, mask) deterministic
        if a=0: break
        else apply_move(s, Move_a) on sim
    → trace_rl, hourly_stats_rl

[ξ + trace → Oracle 행]  (MDP 상태 s의 집계)
  hourly_stats: assign_snapshot, hourly_produce, util_rate
  trace: applied_moves per hour
  → PLAN_ACHV rows (task×hour)
  → ASSIGN rows (eqp×hour, UPH split)
  → CONV rows (batch change moves)
  + guide_rows (상위 alloc)

  result JSON + DB write
